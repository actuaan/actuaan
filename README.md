## 📦 Current Personal Project  
### **Actuarial Life Engine (Private Python Library – In Development)**  
I am currently developing a **proprietary Python library for life actuarial calculations**.  
The goal is to provide a robust, reusable toolset for actuarial teams and insurers.

**What will be included (public API only):**
- **Life contingencies & actuarial factors**: functions to compute survival probabilities, annuities, insurances, endowments and derived actuarial factors (ax, Ax, tpx, tqx), supporting discrete and continuous modes and keyword-only probability APIs.
- **Commutation functions**: efficient vectorized computation of commutation arrays (Dx, Nx, Tx, Sx, etc.) with utilities to derive PVs and reserves.
- **Deterministic cash‑flow modelling**: builders/evaluators and insterest helpers for scheduled flows, timing/frequency adjustments and PV/NPV computations.
- **Premiums & reserves**: net/level/single premium routines and prospective/retrospective reserve calculations with configurable rounding/decimals policies.
- **Mortality tables & interpolation**: built‑in table payloads, table loaders/savers, graduated-rate helpers and interpolation utilities (linear, UDD, configurable lx interpolation) with raw/rounded lx caches.
- **Interest rate utilities & deterministic term‑structures**: conversion helpers (spot↔nominal↔discount), frequency adjustments and scenario helpers for deterministic pricing.
- **IO, persistence & reproducibility**: atomic .ltk persistence, integrity hashes, deterministic save/load and project-local tables_path (defaults to current working directory; configurable via Config/lactuca_config.toml).
- **Public integration helpers**: small-surface APIs for embedding (Config singleton, Tables/TableEntry, TableBuilder, TableSource, LifeTable) intended for integration into pricing engines or core systems.
- **Validation, numerical stability & precision**: centralized normalization (float64), log‑domain techniques for product-of-probabilities, input validators and consistent rounding strategy.

> 🔒 The **source code remains private** because the project is intended for commercial use.  
> 📄 A **public documentation repository** will be published with examples, API, and roadmap.
